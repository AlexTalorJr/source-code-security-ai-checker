## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, label-to-input gap, badge padding vertical |
| sm | 8px | Compact element spacing, table cell padding, input padding, badge padding horizontal |
| md | 16px | Default element spacing, form field vertical gap, button horizontal padding |
| lg | 24px | Section padding, content area padding, navbar horizontal padding, nav link gap |
| xl | 32px | Card padding (login card, 403 card), major form section breaks |
| 2xl | 48px | Empty state vertical padding |
| 3xl | 64px | Not used in this phase |

Exceptions: Login button height is 44px (touch target compliance, matches existing login.html.j2 convention).

## Page Specifications

### Login Page (`login.html.j2` -- REWRITE)

**Layout:** Standalone page (no base.html.j2 navbar). Centered card on `--color-surface` background. Does NOT require authentication.

**Fields:**
1. Username: `<input type="text">` with label "Username", placeholder "Enter your username"
2. Password: `<input type="password">` with label "Password", placeholder "Enter your password"
3. Submit: `<button type="submit">` with text "Log In", full-width, 44px height, accent background

**States:**
- Default: Empty form
- Error: `.error-alert` above form with text "Invalid username or password. Please try again."
- Loading: Button text changes to "Logging in..." and `disabled` attribute set

### 403 Forbidden Page (`403.html.j2` -- NEW)

**Layout:** Standalone page (no base.html.j2 navbar). Centered card on `--color-surface` background, max-width 480px.

**Content:**
- Heading: "Access Denied" (20px, semibold)
- Body: "You need the **{required_role}** role to access this page. Your current role is **{user_role}**." (14px, regular)
- Subtext: "Contact your administrator for access." (14px, `--color-text-secondary`)
- Action: Link "Back to Dashboard" styled as `.btn .btn--accent`, links to `/dashboard/history`

### User Management Page (`users.html.j2` -- NEW, admin only)

**Layout:** Extends `base.html.j2`. Page heading "Users" with "Create User" button (`.btn .btn--accent`) right-aligned in heading row.

**Table columns:**
| Column | Width | Content |
|--------|-------|---------|
| Username | auto | Plain text |
| Role | 100px | `.badge` with role-specific color |
| Created | 140px | Date formatted as `YYYY-MM-DD` |
| Status | 80px | "Active" in green or "Inactive" in gray |
| Actions | 120px | "Edit" link, "Deactivate" button (`.btn--destructive .btn--small`) |

**Empty state:** Heading "No users yet", body "Create your first user to get started."

**Create User form:** Modal or inline form above table (inline preferred for simplicity):
- Username: text input, required
- Password: password input, required, minimum 8 characters
- Role: select dropdown with options: admin, scanner, viewer (default: viewer)
- Submit: "Create User" button (accent)
- Discard: "Discard" link (plain text)

**Edit User form:** Same inline pattern:
- Username: text input, pre-filled, required
- Password: password input, optional (blank = no change), placeholder "Leave blank to keep current"
- Role: select dropdown, pre-selected
- Submit: "Update User" button (accent)

### Token Management Page (`tokens.html.j2` -- NEW, any authenticated user)

**Layout:** Extends `base.html.j2`. Page heading "API Tokens" with "Generate Token" button (`.btn .btn--accent`) right-aligned.

**Table columns:**
| Column | Width | Content |
|--------|-------|---------|
| Name | auto | Plain text (user-provided label) |
| Token | 200px | Masked display: `nvsec_a1b2...` in mono font, `--color-text-secondary` |
| Created | 140px | Date formatted as `YYYY-MM-DD` |
| Expires | 140px | Date or "Never" |
| Last Used | 140px | Date or "Never" |
| Actions | 80px | "Revoke" button (`.btn--destructive .btn--small`) |

**Token creation flow:**
1. User clicks "Generate Token"
2. Inline form appears: Name (text input, required), Expiration (select: "Never", "30 days", "90 days", "1 year")
3. User submits; server returns raw token
4. `.success-alert` appears with: "Token created successfully. Copy it now -- you will not see it again."
5. Below alert: `.token-value` block showing full `nvsec_...` token in mono font
6. User dismisses or navigates; token is masked from this point forward

**Empty state:** Heading "No API tokens", body "Generate a token to authenticate API requests from CI/CD pipelines."

**Soft limit display:** When user has 10 active tokens, "Generate Token" button is disabled with title attribute "Maximum of 10 active tokens reached. Revoke an existing token first."

### Navbar Modifications (`base.html.j2`)

**Left section (nav links):**
- "Scan History" (existing)
- "Trends" (existing)
- "Users" (visible only if `user.role == "admin"`)
- "API Tokens" (visible to all authenticated users)

**Right section:**
- Username text (14px, regular)
- Role badge (10px, semibold, uppercase, role-colored, 4px 8px padding)
- "Log Out" link (existing styling)

**Layout:** `display: flex; align-items: center; gap: 8px;` for the user info group.

## Interaction Contracts

### Form Submission

All forms use standard HTML `<form method="POST">` with server-side validation. No JavaScript required for core functionality.

| Form | Action | Method | Success | Error |
|------|--------|--------|---------|-------|
| Login | `/dashboard/login` | POST | Redirect to `/dashboard/history` | Re-render login with `.error-alert` |
| Create User | `/dashboard/users` | POST | Redirect to `/dashboard/users` with success flash | Re-render form with `.error-alert` |
| Edit User | `/dashboard/users/{id}` | POST | Redirect to `/dashboard/users` | Re-render form with `.error-alert` |
| Deactivate User | `/dashboard/users/{id}/deactivate` | POST | Redirect to `/dashboard/users` | Re-render with `.error-alert` |
| Generate Token | `/dashboard/tokens` | POST | Re-render page with success alert + raw token | Re-render with `.error-alert` |
| Revoke Token | `/dashboard/tokens/{id}/revoke` | POST | Redirect to `/dashboard/tokens` | Re-render with `.error-alert` |

### Confirmation Patterns

| Action | Confirmation Approach |
|--------|----------------------|
| Deactivate User | Browser `confirm()` dialog: "Deactivate user {username}? They will no longer be able to log in." |
| Revoke Token | Browser `confirm()` dialog: "Revoke token {name}? This cannot be undone. Any CI/CD pipelines using this token will stop working." |

### Navigation Guards

| Current Role | Visible Nav Links | Hidden Nav Links |
|-------------|-------------------|------------------|
| admin | Scan History, Trends, Users, API Tokens | none |
| viewer | Scan History, Trends, API Tokens | Users |
| scanner | (no dashboard nav -- scanner role is API-only) | all |

Direct URL access to role-restricted pages redirects to 403 page.
