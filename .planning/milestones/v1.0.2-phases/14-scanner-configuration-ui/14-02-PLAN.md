**JavaScript (in a `<script>` tag at end of content block):**

Implement these functions:

`switchTab(tab)`:
- Toggle `tab-btn--active` class on tab buttons.
- Show/hide `tab-cards` and `tab-yaml` (set display block/none, add/remove `yaml-container--active`).
- If switching to yaml: fetch GET /api/config/yaml (with credentials: 'same-origin'), set CodeMirror value.
- If switching to cards: reload page (simplest approach -- `location.reload()` to get fresh server-rendered data).

`setEnabled(scannerName, value, btn)`:
- Optimistic UI: immediately update toggle button classes in the parent `.toggle-group`.
- Remove all `toggle-btn--on/auto/off` classes from siblings, add appropriate class to clicked button.
- `fetch('/api/config/scanners/' + scannerName, { method: 'PATCH', headers: {'Content-Type': 'application/json'}, credentials: 'same-origin', body: JSON.stringify({enabled: value}) })`.
- On error: revert UI (reload page).

`toggleCard(event, scannerName)`:
- If click target is a toggle button or input, return (don't toggle).
- Close all other open `.card-settings--open` first (accordion).
- Toggle `card-settings--open` class on `#settings-{scannerName}`.

`saveSettings(scannerName)`:
- Read timeout from `#timeout-{scannerName}` input (parseInt).
- Read extra_args from `#args-{scannerName}` input (split on whitespace, filter empty).
- Client-side validation: timeout 30-900, check balanced quotes in args string.
- If invalid: show error in `#card-alert-{scannerName}` as `error-alert`.
- If valid: PATCH /api/config/scanners/{scannerName} with `{timeout, extra_args}`.
- On 200: show `success-alert` "Scanner settings updated successfully." in `#card-alert-{scannerName}`, collapse card after 1500ms.
- On error: show `error-alert` with server error message.

`saveYaml()`:
- Get CodeMirror value.
- PUT /api/config/yaml with body as raw text, `Content-Type: text/plain`.
- On 200: show `success-alert` "Configuration saved successfully." in `#yaml-alert`.
- On 422: show `error-alert` with server validation errors in `#yaml-alert`.
- On other error: show `error-alert` "Failed to save configuration: {message}. Check file permissions on config.yml."

CodeMirror initialization (after DOM ready):
```javascript
var editor;
document.addEventListener('DOMContentLoaded', function() {
    var textarea = document.getElementById('yaml-editor');
    if (textarea && typeof CodeMirror !== 'undefined') {
        editor = CodeMirror.fromTextArea(textarea, {
            mode: 'yaml',
            theme: 'material-darker',
            lineNumbers: true,
            tabSize: 2,
            indentWithTabs: false,
        });
        editor.setSize(null, '60vh');

        // Debounced YAML validation on change
        var validateTimer;
        editor.on('change', function() {
            clearTimeout(validateTimer);
            validateTimer = setTimeout(validateYamlSyntax, 300);
        });
    }
});
```

`validateYamlSyntax()`:
- Try to parse YAML client-side: use a simple check -- if CodeMirror reports no errors, show "YAML valid" in `#yaml-status` with `validation-bar--valid` class.
- Since we don't have a client-side YAML parser, do a lightweight server-side check: POST to PUT /api/config/yaml with a dry-run approach is complex. Instead, just enable the save button always and let server validate. Show neutral status "Edit YAML configuration below" until save attempt.
- Actually, simplest approach: the validation-bar just shows the last save result. On change, clear the status to neutral. On save, show result.

`showAlert(containerId, message, type)`:
- Helper to show alert in a container div. Type is 'success' or 'error'.
- Set innerHTML to `<div class="${type}-alert">${message}</div>`.
- Set display to block.
- If type is 'success', auto-hide after 3000ms.
