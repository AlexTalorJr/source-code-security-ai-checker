## Critical Integration Gap

### cargo-audit tool_name / config key mismatch

**Affects:** SCAN-04, PLUG-02
**Severity:** Critical — blocks all Rust project scans

`CargoAuditAdapter.tool_name` returns `"cargo-audit"` (hyphen) but `config.yml` key is `"cargo_audit"` (underscore). The orchestrator at `orchestrator.py:196` does `settings.scanners[adapter.tool_name]` which raises `KeyError: 'cargo-audit'`.

**Why tests didn't catch it:** Unit tests mock the registry and use adapter names that match their config keys. The mismatch only manifests in the live orchestrator-to-config wiring path.

**Fix options:**
- (a) Change `CargoAuditAdapter.tool_name` to `"cargo_audit"` — aligns with config key convention
- (b) Change `config.yml` key from `cargo_audit` to `cargo-audit` — aligns with CLI tool name
