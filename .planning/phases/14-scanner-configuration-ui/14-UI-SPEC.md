## Component Inventory

### 1. Tab Bar (Cards / YAML Editor)

- Two tabs: "Scanners" (default active) and "YAML Editor"
- Active tab: `--color-accent` bottom border (2px), `--color-text` text
- Inactive tab: no bottom border, `--color-text-secondary` text
- Tab container: bottom border `1px solid --color-border`
- Tabs are `<button>` elements (not links) — switching is client-side JS, no page reload
- On tab switch: the newly active tab fetches fresh config state (Cards tab reloads scanner data via fetch, YAML tab reloads config.yml content via fetch)

### 2. Scanner Card Grid

- CSS Grid: `grid-template-columns: repeat(auto-fill, minmax(320px, 1fr))`
- Grid gap: 24px (--lg)
- Each card: `background: #ffffff`, `border: 1px solid --color-border`, `border-radius: 8px`, `padding: 16px`

### 3. Scanner Card (collapsed)

- **Title row:** Scanner name (14px, weight 600) + status badge (right-aligned)
- **Status badge:** "LOADED" in `--color-success` or "ERROR" in `--color-destructive`, uses existing `.badge` class (12px, weight 600, uppercase)
- **DAST badge:** Nuclei card only — "DAST" badge in `#6610f2` (purple), all others omit badge (implicitly SAST)
- **Language tags:** Below title, `12px`, `--color-text-secondary`, comma-separated (e.g., "Python, JS, Go")
- **Three-state toggle:** Below language tags, margin-top 12px
- **Load error:** If `load_error` is not null, display error text below toggle in `--color-destructive` at 12px

### 4. Three-State Toggle (On / Auto / Off)

- Segmented control with 3 segments, each 56px wide, 36px tall
- Container: `border: 1px solid --color-border`, `border-radius: 4px`, `overflow: hidden`, `display: inline-flex`
- Segments are `<button>` elements inside a container
- Active segment backgrounds:
  - **On:** `--color-success` (#28a745) with white text
  - **Auto:** `--color-text-secondary` (#6c757d) with white text
  - **Off:** `--color-destructive` (#dc3545) with white text
- Inactive segments: `background: #ffffff`, `color: --color-text-secondary`
- Segment borders: `border-right: 1px solid --color-border` (except last)
- Font: 12px, weight 600, uppercase
- On click: POST to API endpoint, update card state without page reload (fetch + DOM update)

### 5. Scanner Card (expanded — inline settings)

- Triggered by clicking card body (not the toggle)
- Expand reveals settings area below toggle with `background: --color-surface`, `padding: 16px`, `border-top: 1px solid --color-border`, `margin: 16px -16px -16px -16px` (flush with card edges)
- **Timeout field:** Label "Timeout (seconds)" (12px, weight 600), `<input type="number" min="30" max="900" step="1">`, uses `.form-input` class
- **Extra args field:** Label "Extra Arguments" (12px, weight 600), `<input type="text" placeholder="--flag1 --flag2 value">`, uses `.form-input` class, `font-family: var(--font-mono)`
- **Save button:** "Save Settings" using `.btn .btn--accent`, right-aligned
- **Validation errors:** Displayed above save button in `--color-destructive` at 12px if timeout out of range or extra_args malformed
- Collapse by clicking card body again (toggle behavior)

### 6. YAML Editor Tab

- CodeMirror 5 loaded via CDN: `codemirror@5.65.18` from `cdnjs.cloudflare.com`
  - Core: `codemirror.min.js` + `codemirror.min.css`
  - Mode: `yaml.min.js`
  - Theme: `material-darker.min.css` (dark editor on light page — visual distinction)
  - Addons: none (keep minimal)
- Editor container: `border: 1px solid --color-border`, `border-radius: 4px`, `overflow: hidden`
- Editor height: `min-height: 400px`, `max-height: 70vh`, scrollable
- Below editor: validation status bar + save button row
- **Validation status bar:** `padding: 8px 16px`, `font-size: 12px`
  - Valid: `--color-success` text "YAML valid"
  - Invalid: `--color-destructive` text showing first error (e.g., "Line 15: mapping values are not allowed here")
  - Pydantic error: `--color-destructive` text (e.g., "scanners.semgrep.timeout: value is not a valid integer")
- **Save button:** "Save Config" using `.btn .btn--accent`, right-aligned, disabled (opacity 0.5, cursor not-allowed) when validation fails
- On save success: `success-alert` appears above editor ("Configuration saved successfully")
- On save failure: `error-alert` appears above editor with server error message

### 7. Navbar Addition

- New link "Scanners" added between "Trends" and "Users" in nav
- Visible only to admin users (same `{% if user and user.role == "admin" %}` guard as Users link)
- Active state: `navbar__link--active` when `active_page == "scanners"`

## Interaction Contracts

### Tab Switching
- Client-side JavaScript, no page reload
- Both tabs share a single page template (`scanners.html.j2`)
- Active tab content visible via `display: block/none`
- On tab switch: fetch fresh data from API (Cards: GET scanner list + config; YAML: GET raw config.yml content)

### Three-State Toggle
- Click a segment to activate it
- Sends PATCH request to config API with `{scanner_name: {enabled: "auto"|true|false}}`
- Optimistic UI: update toggle visually immediately, revert on API error
- No page reload

### Card Expand/Collapse
- Click card body (not toggle) to expand/collapse settings
- Only one card expanded at a time (accordion behavior)
- Expand: slide-down animation via `max-height` transition (200ms ease)
- Collapse: slide-up animation (200ms ease)

### Settings Save (Card)
- Validate timeout: integer, 30-900 range
- Validate extra_args: balanced quotes, no empty segments after split
- On valid: POST to API, show success-alert inside card, collapse after 1.5s
- On invalid: show validation error below fields, do not send request

### YAML Save
- Real-time YAML syntax validation on CodeMirror `change` event (debounced 300ms)
- On save click: send full YAML string to API for Pydantic validation + file write
- Server returns 200 on success, 422 with validation errors on failure
- Disable save button while validation errors are present

### Concurrent Edit Safety
- No locking mechanism (single-admin expected usage)
- YAML editor loads fresh content on tab switch (prevents stale edits)
- Card tab reloads scanner states on tab switch
