<objective>
Implement all five scanner adapters (Semgrep, cppcheck, Gitleaks, Trivy, Checkov) that each run their respective tool as a subprocess, parse the structured output (JSON or XML), and return normalized `list[FindingSchema]`. Each adapter inherits from `ScannerAdapter` ABC.

Purpose: These adapters are the core scanning engine -- each one bridges a security tool's native output to the unified finding model.
Output: Five working adapters with comprehensive unit tests using fixture data.

  - Return list of FindingSchema

**cppcheck adapter** (`src/scanner/adapters/cppcheck.py`):
- Class `CppcheckAdapter(ScannerAdapter)` with `tool_name = "cppcheck"`
- `_version_command()`: returns `["cppcheck", "--version"]`

  - Append extra_args if provided
  - Call `self._execute(cmd, timeout)` -- NOTE: cppcheck writes XML to STDERR, not stdout

  - For each `<error>` with at least one `<location>`, create FindingSchema
  - Severity map: `{"error": Severity.HIGH, "warning": Severity.MEDIUM, "style": Severity.LOW, "performance": Severity.LOW, "portability": Severity.LOW, "information": Severity.INFO}`
  - Skip errors with id `"missingIncludeSystem"` (noisy, not security-relevant)
  - Return list of FindingSchema
