# Phase 4 — UI Design Contract

> Visual and interaction contract for generated HTML and PDF security scan reports. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Note:** This phase produces server-rendered report artifacts (self-contained HTML file + PDF), not a React/SPA frontend. The design contract covers Jinja2 HTML template styling, PDF layout via WeasyPrint, and chart generation via matplotlib. There is no component library or shadcn involvement.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (server-rendered Jinja2 templates) |
| Preset | not applicable |
| Component library | none (vanilla HTML/CSS/JS) |
| Icon library | none (Unicode symbols for badges and indicators) |
| Font | system-ui stack: `-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif` |

## Copywriting Contract

### HTML Report

| Element | Copy |
|---------|------|
| Page title | `Security Scan Report - {branch_or_path}` |
| Gate passed banner | `QUALITY GATE: PASSED` |
| Gate failed banner | `QUALITY GATE: FAILED` |
| Gate fail reasons prefix | `Blocked by:` followed by bulleted list of reasons |
| Delta section heading | `Changes Since Last Scan` |
| Delta badges | `{N} new` / `{N} fixed` / `{N} persisting` |
| No delta available | `First scan on this branch -- no previous scan to compare.` |
| Empty severity group | Do not render the section at all (skip empty groups) |
| Sidebar heading | `Filters` |
| Sidebar group headings | `Severity` / `Tool` / `Component` |
| Finding card: AI fix label | `AI Fix Suggestion` |
| Finding card: no AI fix | `AI analysis not available for this finding.` |
| Code context label | `Code Context` (above the code block) |
| Compound risks heading | `Compound Risks` |
| Compound risk: no items | Do not render the section at all |
| Metadata section heading | `Scan Details` |
| Metadata labels | `Date:` / `Branch:` / `Commit:` / `Duration:` / `Tool Versions:` / `AI Cost:` |
| Metadata: commit unknown | `N/A` |
| Footer | `Generated by aipix-security-scanner on {date}` |

### PDF Report

| Element | Copy |
|---------|------|
| Title | `Security Scan Report` |
| Subtitle | `{branch_or_path} -- {date}` |
| Gate passed | `Quality Gate: PASSED` |

| Executive summary heading | `Executive Summary` |
| Executive summary body | `This scan analyzed {target} and identified {total} finding(s) across {tool_count} security tools. {critical_count} Critical and {high_count} High severity issues were found. {compound_count} compound risk(s) were identified through AI correlation.` |
| Severity chart title | `Severity Distribution` |
| Tool chart title | `Findings by Tool` |
| Findings table heading | `Finding Details` |
| Findings table columns | `Severity` / `File` / `Rule` / `Description` |
| Delta section heading | `Changes Since Last Scan` |

| Compound risks heading | `Compound Risks (AI-Identified)` |

| Element | Copy |
|---------|------|
| Delta summary line | `Delta: +{new} new, -{fixed} fixed, {persisting} persisting` |

## Layout Contract

### HTML Report Structure

```
+--------------------------------------------------+
| GATE STATUS BANNER (full width, prominent)       |
|   PASSED/FAILED + fail reasons if any            |
+--------------------------------------------------+
| DELTA SUMMARY BAR                                |
|   [N new] [N fixed] [N persisting]               |
+--------+-----------------------------------------+
| SIDEBAR | MAIN CONTENT                           |
|         |                                        |
| Filters | SCAN METADATA (date, branch, etc.)     |
|         |                                        |
| [x] Sev | COMPOUND RISKS (if any)               |
|   Crit  |   Risk cards with severity + title     |
|   High  |                                        |
|   Med   | CRITICAL FINDINGS (N)                  |
|   Low   |   Finding card                         |
|   Info  |   Finding card                         |
|         |                                        |
| [x] Tool| HIGH FINDINGS (N)                      |
|   Semg  |   Finding card                         |
|   cpp   |                                        |
|   Gitl  | MEDIUM FINDINGS (N)                    |
|   Trivy |   ...                                  |
|   Check |                                        |
|         | LOW FINDINGS (N)                       |
| [x] Comp|   ...                                  |
|   comp1 |                                        |
|   comp2 | INFO FINDINGS (N)                      |
|         |   ...                                  |
+--------+-----------------------------------------+
| FOOTER                                           |
+--------------------------------------------------+
```

- Sidebar width: 240px fixed
- Sidebar position: sticky (scrolls with page but stays visible)
- Main content: fluid, fills remaining width
- Gate banner: 100% width, 48px min-height, centered text
- Finding cards: full width of main content area, 16px padding, 1px solid border, 8px border-radius
- Max content width: none (fills available space)
- Minimum viewport support: 1024px (report is for desktop viewing)

### Finding Card Layout

```
+--------------------------------------------------+
| [SEVERITY BADGE] [TOOL TAG] [COMPONENT TAG]     |
| Rule: rule_id                                     |
| File: path/to/file.py:42-48                      |
+--------------------------------------------------+
| Description text                                  |
+--------------------------------------------------+
| Code Context                                      |
| +----------------------------------------------+ |
| | 42 |   vulnerable_code_here()                 | |
| | 43 |   more_code()                            | |
| +----------------------------------------------+ |
+--------------------------------------------------+
| AI Fix Suggestion                                 |
| +---------------------+------------------------+ |
| | Before              | After                  | |
| | - old_code()        | + fixed_code()         | |
| +---------------------+------------------------+ |
+--------------------------------------------------+
```

- Severity badge: inline-block, 4px vertical padding, 8px horizontal padding, 4px border-radius, white text on severity color background, 12px semibold uppercase
- Tool/component tags: inline-block, 4px vertical padding, 8px horizontal padding, 4px border-radius, `#f8f9fa` background, `#212529` text, 12px regular
- Code block: `#f8f9fa` background, 12px padding, 4px border-radius, monospace 13px, line numbers in `#6c757d`, overflow-x auto
- Diff block: side-by-side layout using CSS grid (2 equal columns), removed lines with `#f8d7da` background, added lines with `#d4edda` background, monospace 13px
- New finding indicator: 4px left border in `#dc3545` on the finding card (only when delta available and finding is new)

### PDF Report Structure

```
Page 1:
+--------------------------------------------------+
| TITLE + SUBTITLE                                  |
| GATE STATUS                                       |
| EXECUTIVE SUMMARY (2-3 sentences)                |
| DELTA SUMMARY (1 line)                           |
+--------------------------------------------------+
| [PIE CHART]          [BAR CHART]                 |
| Severity Distribution  Findings by Tool          |
+--------------------------------------------------+

Pages 2-4:
+--------------------------------------------------+
| FINDINGS TABLE                                    |
| Severity | File | Rule | Description             |
|----------|------|------|-------------------------|
| Critical | ...  | ...  | ...                     |
| High     | ...  | ...  | ...                     |
| ...      | ...  | ...  | ...                     |
+--------------------------------------------------+
| COMPOUND RISKS TABLE (if any)                    |
| Severity | Title | Related Findings              |
+--------------------------------------------------+

Every page:
+--------------------------------------------------+
| FOOTER: scanner name + page N of M               |
+--------------------------------------------------+
```

- PDF page size: A4 (210mm x 297mm)
- Margins: 20mm all sides
- Charts: 80mm x 80mm each, placed side by side on page 1
- Chart resolution: 150 DPI
- Table: full width, alternating row backgrounds (`#ffffff` / `#f8f9fa`), 1px `#dee2e6` borders
- Findings sorted: Critical first, then High, Medium, Low, Info within the table

## Interaction Contract

### HTML Report Sidebar Filters

| Interaction | Behavior |
|-------------|----------|
| Check/uncheck severity checkbox | Instantly show/hide all findings of that severity. Update visible count in each severity group heading. |
| Check/uncheck tool checkbox | Instantly show/hide findings from that tool. Applies as AND with severity filter. |
| Check/uncheck component checkbox | Instantly show/hide findings from that component. Applies as AND with other filters. |
| All filters unchecked in a group | Show nothing (strict AND logic, not "show all") |
| Page load default | All checkboxes checked, all findings visible |
| Badge counts | Each checkbox label shows count of matching findings: `Critical (5)`. Counts update as other filter groups change. |
| Empty severity section after filter | Hide the entire severity section heading when zero findings visible in that group |

### No Other Interactions

The HTML report is a static document with client-side filtering only. There are no:
- Forms or data submission
- Navigation between pages (single page, scroll)
- Modals or dialogs
- Tooltips (keep it simple)
- Animations or transitions
- External network requests (fully offline-capable)
