    2. Create `Dockerfile`:
       ```dockerfile
       FROM python:3.12-slim AS base

       # System deps (curl for healthcheck, git for Phase 2 repo cloning)
       RUN apt-get update && apt-get install -y --no-install-recommends \
           curl \
           && rm -rf /var/lib/apt/lists/*

       # Non-root user for security
       RUN groupadd -r scanner && useradd -r -g scanner -d /app scanner
       RUN mkdir -p /data && chown scanner:scanner /data

       WORKDIR /app

       # Python deps first (Docker cache layer)
       COPY pyproject.toml .
       RUN pip install --no-cache-dir .

       # App code
       COPY src/ src/
       COPY alembic.ini .
       COPY alembic/ alembic/
       COPY config.yml.example config.yml

       # Set ownership
       RUN chown -R scanner:scanner /app

       USER scanner

       EXPOSE 8000

       CMD ["uvicorn", "scanner.main:app", "--host", "0.0.0.0", "--port", "8000"]
       ```

       Key decisions:
       - python:3.12-slim base (not alpine -- avoids musl compilation issues with C extensions)
       - Non-root user "scanner" per RESEARCH.md Pitfall 5
       - /data directory owned by scanner user for SQLite writes
       - config.yml.example copied as default config.yml (no secrets, just defaults)
       - pyproject.toml copied first for Docker layer caching

    3. Create `docker-compose.yml`:
       ```yaml
       services:
         scanner:
           build: .
           ports:
             - "${SCANNER_PORT:-8000}:8000"
           volumes:
             - scanner_data:/data
             - ./config.yml:/app/config.yml:ro
           environment:
             - SCANNER_DB_PATH=/data/scanner.db
             - SCANNER_API_KEY=${SCANNER_API_KEY:-}
             - SCANNER_CLAUDE_API_KEY=${SCANNER_CLAUDE_API_KEY:-}
             - SCANNER_CONFIG_PATH=/app/config.yml
           healthcheck:
             test: ["CMD", "curl", "-f", "http://localhost:8000/api/health"]
             interval: 30s
             timeout: 10s
             retries: 3
             start_period: 10s
           restart: unless-stopped

       volumes:
         scanner_data:
       ```

       Key decisions:
       - No `version:` key (deprecated in modern Docker Compose)
       - scanner_data named volume for /data (SQLite persistence across restarts)
       - config.yml bind mount with :ro (read-only)
       - Secrets via environment variables only (SCANNER_API_KEY, SCANNER_CLAUDE_API_KEY)
       - Healthcheck using curl to /api/health
       - start_period: 10s gives app time to boot before first health check
  </action>
