# Руководство пользователя

## Что такое aipix-security-scanner?

Инструмент сканирования безопасности, который анализирует исходный код на наличие уязвимостей с помощью пяти параллельных инструментов статического анализа, обогащает результаты ИИ-анализом через Claude и формирует практические отчёты с рекомендациями по исправлению.

## Запуск сканирования

### Через API

```bash
curl -X POST http://localhost:8000/api/scans \
  -H "X-API-Key: $SCANNER_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"repo_url": "https://github.com/your-org/repo.git", "branch": "main"}'
```

API немедленно возвращает идентификатор сканирования (202 Accepted). Сканирование выполняется асинхронно в фоновой очереди.

### Через CLI

```bash
scanner scan --repo-url https://github.com/your-org/repo.git --branch main
```

CLI выполняет сканирование напрямую и выводит результаты в stdout. Используйте `--format html` или `--format pdf` для генерации файлов отчётов.

### Через панель управления

Перейдите по адресу `http://localhost:8000/dashboard`, введите URL репозитория и ветку, затем отправьте форму. Панель управления отображает ход сканирования и результаты на месте.

## Уровни серьёзности

| Уровень | Значение | Действие |
|---------|----------|----------|
| **CRITICAL** | Непосредственный риск эксплуатации (например, SQL-инъекция, RCE) | Немедленное исправление; блокирует развёртывание |
| **HIGH** | Серьёзная уязвимость (например, обход аутентификации, захардкоженные секреты) | Исправить до релиза |
| **MEDIUM** | Умеренный риск (например, слабое шифрование, отсутствие заголовков) | Исправить в текущем спринте |
| **LOW** | Незначительная проблема (например, подробные сообщения об ошибках) | Исправить при возможности |
| **INFO** | Информационная находка (например, использование устаревшего API) | Ознакомиться, действий не требуется |

## Отчёты

### HTML-отчёты

Интерактивные HTML-отчёты включают:

- **Раздел сводки** -- общее количество находок, распределение по серьёзности, результат шлюза качества
- **Фильтруемая таблица находок** -- фильтрация по серьёзности, инструменту, пути к файлу
- **Контекст кода** -- фрагменты исходного кода с выделением уязвимых строк
- **ИИ-рекомендации по исправлению** -- сгенерированный Claude код исправления с пояснениями
- **Составные риски** -- кросс-инструментальные корреляционные находки, выявленные ИИ
- **Диаграммы** -- круговая диаграмма распределения по серьёзности и столбчатая диаграмма находок по инструментам

Доступ к HTML-отчётам через `GET /api/scans/{id}/report/html` или из панели управления.

### PDF-отчёты

PDF-отчёты предоставляют формальный документ для руководства:

- **Краткое содержание** -- метаданные сканирования, счётчики по серьёзности, результат шлюза
- **Диаграммы** -- встроенные PNG-диаграммы (распределение по серьёзности, по инструментам)
- **Детальные находки** -- сгруппированные по серьёзности с фрагментами кода
- **Раздел составных рисков** -- выявленные ИИ межкомпонентные уязвимости

Доступ к PDF-отчётам через `GET /api/scans/{id}/report/pdf`.

## Качественный шлюз

Шлюз качества оценивает результаты сканирования по настроенным порогам серьёзности. По умолчанию любая находка уровня CRITICAL или HIGH приводит к непрохождению шлюза.

- **pass** -- нет находок на уровне или выше настроенного порога серьёзности
- **fail** -- одна или более находок на уровне или выше порога, либо составные риски с серьёзностью Critical/High при включённом параметре `include_compound_risks`

Результаты шлюза качества доступны через `GET /api/scans/{id}/gate` и отображаются в отчётах и панели управления.

## ИИ-анализ

Каждая партия находок отправляется Claude для контекстного анализа:

- **Контекстный обзор** -- понимание назначения кода и определение истинности находки
- **Рекомендации по исправлению** -- конкретные изменения кода для устранения уязвимости
- **Составные риски** -- выявление цепочек атак, охватывающих несколько находок (например, обход аутентификации + IDOR = захват учётной записи)

Стоимость ИИ-анализа на сканирование отслеживается и ограничивается параметром `ai.max_cost_per_scan` в конфигурации.

## Дельта-сравнение

Когда репозиторий сканировался ранее, сканер автоматически вычисляет дельту:

- **Новые находки** -- уязвимости, отсутствовавшие в предыдущем сканировании
- **Исправленные находки** -- уязвимости из предыдущего сканирования, которых больше нет
- **Сохраняющиеся находки** -- уязвимости, присутствующие в обоих сканированиях

Дельта вычисляется путём сопоставления fingerprint-ов между текущим и последним предыдущим сканированием того же репозитория и ветки. Первое сканирование не возвращает дельту (нет предыдущей базовой линии).

## Управление ложными срабатываниями

### Через панель управления

В представлении находок нажмите кнопку подавления на любой находке. Укажите причину подавления.

### Через API

```bash
curl -X POST http://localhost:8000/api/suppressions \
  -H "X-API-Key: $SCANNER_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"fingerprint": "<finding-fingerprint>", "reason": "False positive: test fixture"}'
```

Подавленные находки исключаются из оценки шлюза качества и помечаются в отчётах.
